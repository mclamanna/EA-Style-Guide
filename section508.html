Section 508 and Web Accessibility

Section 508 is a federal law that requires all electronic and information technology developed, procured, maintained, or used by the federal government be accessible to people with disabilities. The U.S. Department of Justice Office of Civil Rights is charged with enforcing Section 508.

We are responsible for ensuring the information and services we provide can be used as effectively by people with disabilities as by those without.

To review the Section 508 standards, refer to: www.section508.gov

Web Accessibility Tutorials:

https://www.w3.org/WAI/tutorials/

This collection of tutorials shows you how to develop web content that is accessible to people with disabilities, and that provides a better user experience for everyone.

Designing for Web Accessibility

Basic steps to make your user interface design and visual design accessible to people with disabilities.

  • Provide sufficient contrast between foreground and background

    • Foreground text needs to have sufficient contrast with background colors. This includes text on images, background gradients, buttons, and other elements.

    Insufficient

    Some people cannot read text if there is not sufficient contrast between the text and background. For others, bright colors (high luminance) are not readable; they need low luminance.

    Sufficient

    Some people cannot read text if there is not sufficient contrast between the text and background. For others, bright colors (high luminance) are not readable; they need low luminance.

      Requirements:

    1. Contrast ratio is at least

      4.5:1

      for normal size text (less than 18 point if not bold and less than 14 point if bold)
    2. Contrast ratio is at least

      3:1

      for large size text (at least 18 point if not bold and at least 14 point if bold)

  • Don’t use color alone to convey information

    • When using color to differentiate elements, also provide additional identification that does not rely on color perception.

    Color only
    • Which is the right-angled triangle?



    Color and number
    • Which is the right-angled triangle?



  • Ensure that interactive elements are easy to identify

    • Provide distinct styles for interactive elements, such as links and buttons, to make them easy to identify. For example, change the appearance of links on mouse hover, keyboard focus, and touch-screen activation. Ensure that styles and naming for interactive elements are used consistently.

  • Provide clear and consistent navigation options

    • Ensure that navigation across pages within a website has consistent naming, styling, and positioning. Provide more than one method of website navigation, such as a site search or a site map. Help users understand where they are in a website or page by providing orientation cues, such as breadcrumbs and clear headings.

  • Ensure that form elements include clearly associated labels

    • All form fields have to have a descriptive label adjacent to the field. Avoid having too much space between labels and fields.

  • Provide easily identifiable feedback

    • Provide feedback for interactions, such as confirming form submission, alerting the user when something goes wrong.

    • Use whitespace and proximity to make relationships between content more apparent. Style headings to group content, reduce clutter, and make it easier to scan and understand.

Associate a label with every form control

Basic considerations to get started developing web content that is more accessible to people with disabilities.

  • Associate a label with every form control

    • Use a ‘for’ attribute on the <label> element linked to the id attribute of the form element.

    Example: Using for and id attributes
    Rendered
    Code Snippet
    <label for="username"</label>

    <input id="username" type="text" name="username">

  • Include alternative text for images

    • Ensure that alternative text for images is added to all informational and functional images. Use empty alternative text, alt="" for decorative images, or include them in the CSS instead.

  • Use mark-up to convey meaning and structure

    • Use appropriate mark-up for headings, lists, tables, etc. HTML5 provides additional elements, such as <nav> and <aside>, to better structure your content.

  • Make tables accessible

    • Accessible tables need HTML markup that indicates header cells and data cells and defines their relationship. Header cells must be marked up with <th>, and data cells with <td> to make tables accessible. For more complex tables, explicit associations may be needed using scope, id, and headers attributes. Use tutorial (https://www.w3.org/WAI/tutorials/tables/) that shows how to apply appropriate structural markup to tables.

  • Ensure that all interactive elements are keyboard accessible

    • Think about keyboard access, especially when developing interactive elements, such as menus, mouseover information, collapsable accordions, or media players. Use tabindex="0" to add an element that does not normally receive focus, such as <div> or <span>, into the navigation order when it is being used for interaction. Use scripting to capture and respond to keyboard events.

Testing for Accessibility:

First Review (https://www.w3.org/WAI/test-evaluate/preliminary/)

A few simple steps to check for basic 508 compliance

  • Skip navigation

    • Skip to main content link placed at the beginning of the page before the navigation; useful for keyboard navigation.

    What to check for:
    • Check if the link appears first when you start tabbing through the page.

  • Page title

    • Shown in browsers' tabs, search engine results; used for bookmarks, read by screen readers In the web page markup  - <title> within <head>

    What to check for:
    • Check if there is a title that adequately and briefly describes the content.
    • Best practice is for titles to be "front-loaded" with the important and unique info first.

  • Image text alternatives ("alt text") (pictures, illustrations, charts, etc.)

    • Convey the purpose of an image, read by screen readers. Can be tested with WAVE extension.

    What to check for:
    • If an image is required for interacting with or understanding the content, it needs alternative text that convey the same meaning as the image.
    • If an image is for decoration, it should have null alt (alt="").

  • Headings

    • Web pages have sections of information separated by visual headings marked <h1> to <h6>. For keyboard navigation or screen reader heading levels should have a meaningful hierarchy, e.g.:

  • Heading Level 1 <h1>
    • Heading Level 2 <h2>
      • Heading Level 3 <h3>
      • Heading Level 3 <h3>
    • Heading Level 2 <h2>
      • Heading Level 3 <h3>
        • Heading Level 4 <h4>
        • Heading Level 4 <h4>
    • Heading Level 2 <h2>

    • Can be tested with WAVE extension.

    What to check for:
    • Check if there is at least one heading at the page.
    • All text that looks like a heading is marked up as a heading.
    • Ideally the page starts with an "h1" — which is usually similar to the page title — and does not skip levels; however, these are not absolute requirements.

  • Contrast ratio ("color contrast")

    • Some people cannot read text if there is not sufficient contrast between the text and background Can be tested with WAVE extension or Color and Contrast Analyzer (CCA) tool.

    What to check for:
    • Minimum contrast ratio is at least 4.5:1 for normal-size text.

  • Resize text

    • Some people need to enlarge web content in order to read it. Can be tested by changing browser settings – text size, text only zoom, page zoom.

    What to check for:
    • All text gets larger, doesn't disappear, cut off or overlap
    • All interactive elements are visible and usable.

  • Keyboard access and visual focus

    • Many people cannot use a mouse and rely on the keyboard for navigation and interactions. Keyboard focus should be visible and should follow a logical order through the page elements.

    How to check:
    • Click in the address bar, then put your mouse aside and do not use it.
    • Press the 'Tab' key to move through the elements on the page. You can press 'Shift-Tab' to go backwards.
    • To move within elements such as drop-down lists and menu bars, press the arrow keys.
    • To select a specific item within a drop-down list:
      • Tab to the list box
      • Use the arrow keys to move the focus to items
      • When an item has focus, press the Enter key or Space bar to select that item

    What to check for:
    • Tab to all:
      Check that you can tab to all the elements, including links, form fields, buttons, and media player controls.
    • Tab away:
       Check that you can tab away from all elements (no "keyboard traps").
    • Tab order:
       Check that the tab order follows top to bottom, left to right in sequence.
    • Visual focus:
      Check that the focus is clearly visible.
    • All functionality by keyboard:
       Check that you can do everything with the keyboard (no mouse needed).
    • Drop-down lists:
       Check if you can use the arrow keys to move through all the options a drop-down list without triggering an action.
    • Image links:
       Check that when images are links, they have clear visual focus and can be activated by pressing the Enter key.

  • Forms, labels, and errors

    • Users need to be able to interact with forms using keyboard, voice input or screen readers.

    What to check for:
    • Check that all form controls are keyboard accessible including drop-down lists
    • Check if buttons have a descriptive value, form inputs have text labels, related form elements are grouped with fieldset/legend. ARIA labelling is used when necessary.
    • Form validation errors are efficient, intuitive, accessible, and easy to correct.
    • If specific format, value, or length is required, info is provided within the element's label.

  • Moving, Flashing, or Blinking Content

    • Users need to be able to control moving content (including carousel)

    What to check for:
    • Check if there is a way for the user to pause, stop, or hide the movement or control the frequency of the update.

  • Multimedia (video, audio) alternatives

    • Information delivered in audio or video has to be provided in alternative formats.

    What to check for:
    • Check if media player controls are labeled and keyboard accessible.
    • Check if video have captions (subtitles), transcripts and/or audio descriptions
